Overview

The MOBSS API is REST-based, uses JSON Web Tokens for authentication and returns
responses in JSON.

To interact with the API you’ll need an HTTP library, a JSON Parser and your
customer or partner number and password. For the examples in this document, we will be using
Javascript.

Authenticating

The list of endpoints is given below. However, the first step in interacting with the API will be authentication. This involves calling an authenticating endpont with your credentials and receving a JSON Web Token (jwt) back. You can then use this token to call any of the other endpoints. The authentication endpoint is as follows:

POST /api/authentication

e.g. https://localhost/api/authenticate

JSON return should look something like:

{
    "success": true,
    "message": "Enjoy your token!",
    "token": "eyJhbGciOiJIUzI1NiJ9.YWdwYnJ0ZGs.FXyGVuRDjwxVbUiJm6_v7nhBAqaE23LyPKliEla_be4"
}

The parameters for the call are your credentials (issued by MOBSS):

Key Value
pass secretpasswd

The parameters are sent in the body using content-type:application/x-www-form-urlencoded header and the call structure is as follows:

POST /api/authenticate HTTP/1.1
Host: localhost
Cache-Control: no-cache
Content-Type: application/x-www-form-urlencoded

pass=secretpasswd

Errors

When an error occurs with the request, you’ll receive an HTTP respponse and a JSON response containing further detail about the problem,

The following potential errors messages are possible with this endpoint:

{
    "error": "Bad Request",
    "message": "Invalid Credentials",
    "status_code": "400"
}

Authenticating Requests

Once you have your token, just include it in all subsequent endpoint calls. Simply pass the token in the body with the key value token.

token = eyJhbGciOiJIUzI1NiJ9.YWdwYnJ0ZGs.FXyGVuRDjwxVbUiJm6_v7nhBAqaE23LyPKliEla_be4

Errors

The following error is possible when calling all endpoints other than the authentication endpoint, typically as a result of a bad or expired token.

{
    "error": "Unauthorised",
    "message": "Unauthorised request",
    "status_code": "401"
}

Other errors resulting from the sending of invalid datatypes or other types of interal APi error:

{
    "error": "Internal Server Error",
    "message": "Internal API error",
    "status_code": "500"
}

API Endpoints

The following endpoints are available as of this version of the API:

  • POST /api/getPeople
  • POST /api/getEvents
  • POST /api/getAttendance
  • POST /api/getForAttendee
  • POST /api/createInviteList
  • POST /api/addInvitee
  • POST /api/deleteInvitee

The following sections detail instructions for using each of the endpoints.

POST api/getEvents

Allows you to retrieve all the events currently in the MOBSS database, or sub-group of these based on Event ID, Event Name, Event Date or a range of dates:

Parameters

NAME TYPE REQUIRED DESCRIPTION
eventID integer No Return an event. If this request parameter is filled in the others are ignored
eventName string No If eventID is left blank, this returns all events with a particular event name
eventStartDate string No If first two parms left blank, returns events on this date OR denotes start of date range search if eventEndDate also in the request
eventEndDate string No Returns all events in a date range ending with this date (as long as eventStartDate is also sent in the request and first two parms are blank)

JSON response

The following is example JSON returned from a successful call (Status Code = 200)

{

No specific errors for this endpoint.

POST api/createInviteList

Allows you to create an invite list by supplying a name and an optional comment.

Parameters

NAME TYPE REQUIRED DESCRIPTION
listName string Yes Name for the invite list
listComment string No Optional comment for the invite list

JSON response

The following is example JSON returned from a successful call (Status Code = 200). Note that the insertId returned provides the database ID for the Invite List for use in adding and deleting Invitees (see later).

{       
    "fieldCount": 0,
    "affectedRows": 1,
    "insertId": 172,
    "serverStatus": 2,
    "warningCount": 0,
    "message": "",
    "protocol41": true,
    "changedRows": 0
}

Errors

{
    "error": "Bad Request",
    "message": "Invite List Name is Blank",
    "status_code": "400"
}

POST api/addInvitee

Allows you to add invitees to an invite list. Note that the listID is provided in the return from api/createInviteList or through commandCenter.

Parameters

NAME TYPE REQUIRED DESCRIPTION
listID integer Yes Invite List ID
badgeNumber integer Yes Invitee badge number
lastName string Yes Invitee name
firstName string Yes Invitee name
emailAddress string No Invitee email
notificationNumber integer No Contact number for invitee
numberFormat string No Notification number format

JSON response

The following is example JSON returned from a successful call (Status Code = 200). 

{       
   "fieldCount": 0,
    "affectedRows": 1,
    "insertId": 0,
    "serverStatus": 2,
    "warningCount": 0,
    "message": "",
    "protocol41": true,
    "changedRows": 0
}

Errors

{
    "error": "Bad Request",
    "message": "Invite list does not exist",
    "status_code": "400"
}
