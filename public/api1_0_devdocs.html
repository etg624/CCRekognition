Overview

The MOBSS API is REST-based, uses JSON Web Tokens for authentication and returns responses in JSON.

To interact with the API you'll need an HTTP library, a JSON Parser and your customer or partner password. For the examples in this document, we will be using Javascript.

Authenticating

The list of endpoints is given below. However, the first step in interacting with the API will be authentication. This involves calling an authenticating endpoint with your credentials and receiving a JSON Web Token (jwt) back. You can then use this token to call any of the other endpoints. The authentication endpoint is as follows:

POST /api/authenticate

e.g. https://localhost/api/authenticate

This JSON return should look something like:

{
    "success": true,
    "message": "Enjoy your token!",
    "token": "eyJhbGciOiJIUzI1NiJ9.YWdwYnJ0ZGs.FXyGVuRDjwxVbUiJm6_v7nhBAqaE23LyPKliEla_be4"
}

The parameters for the call are your credentials (issued by MOBSS):

Key Value
pass secretpasswd

The parameters are sent in the body using content-type:application/x-www-form-urlencoded header and the call structure is as follows:

POST /api/authenticate HTTP/1.1
Host: localhost
Cache-Control: no-cache
Content-Type: application/x-www-form-urlencoded

pass=secretpasswd

Errors

When an error occurs with the request, you'll receive an HTTP response and a JSON response containing further detail about the problem,

The following potential error messages are possible with this endpoint:

{
    "error": "Bad Request",
    "message": "Invalid Credentials",
    "status_code": "400"
}

Authenticating Requests

Once you have your token, just include it in all subsequent endpoint calls. Simply pass the token in the body with the key value token.

token = eyJhbGciOiJIUzI1NiJ9.YWdwYnJ0ZGs.FXyGVuRDjwxVbUiJm6_v7nhBAqaE23LyPKliEla_be4

Errors

The following error is possible when calling all endpoints other than the authentication endpoint, typically as a result of a bad or expired token.

{
    "error": "Unauthorised",
    "message": "Unauthorised request",
    "status_code": "401"
}

Other errors resulting from the sending of invalid datatypes or other types of interal API error:

{
    "error": "Internal Server Error",
    "message": "Internal API error",
    "status_code": "500"
}

API Endpoints

The following endpoints are available as of this version of the API:

  • Get People [POST /api/getPeople]
  • Get Events [POST /api/getEvents]
  • Get Attendance [POST /api/getAttendance]
  • Get For Attendee [POST /api/getForAttendee]
  • Create Invite List [POST /api/createInviteList]
  • Add Invitee [POST /api/addInvitee]
  • Delete Invitee [POST /api/delInvitee]
  • Mark Attendance [POST /api/markAttendance]
  • Create Event [POST /api/createEvent]

The following sections detail instructions for using each of the endpoints.

Get Events

POST api/getEvents

Allows you to retrieve all the events currently in the MOBSS database, or sub-group of these based on Event ID, Event Name, Event Date or a range of dates:

Parameters

The parameters are sent in the body along with the token, using content-type:application/x-www-form-urlencoded.

NAME TYPE REQUIRED DESCRIPTION
eventID integer No Return an event. If this request parameter is filled in the others are ignored
eventName string No If eventID is left blank, this returns all events with a particular event name
eventStartDate string No If first two parms left blank, returns events on this date OR denotes start of date range search if eventEndDate also in the request
eventEndDate string No Returns all events in a date range ending with this date (as long as eventStartDate is also sent in the request and first two parms are blank)

JSON response

The following is example JSON returned from a successful call (Status Code = 200)
JSON
{
"EventID": 101,
"EventName": "Regulatory Training",
"EventDateTime": "2017-12-05 00:30:00",
"EventLocationName": "",
"EventSponsorName": "feb",
"DurationInMins": "30",
"Latitude": "null",
"Longitude": "null",
"RecordStatus": "null",
"Comments": "",
"updateTime": "1512614159630",
"EventsType": "Meeting",
"InvitationListID": 0,
"TempID": null
}

Errors

No specific errors for this endpoint.

Allows you to create an invite list by supplying a name and an optional comment.

Parameters

The parameters are sent in the body along with the token, using content-type:application/x-www-form-urlencoded.

lastName string Yes Invitee name
firstName string Yes Invitee name
emailAddress string No Invitee email
notificationNumber int No Contact number for invitee
numberFormat string No Notification number format

NAME TYPE REQUIRED DESCRIPTION

JSON response

Create Event

POST api/createEvent

Allows you to create an Event. A successful call returns the Event ID.

Parameters

The parameters are sent in the body along with the token, using content-type:application/x-www-form-urlencoded.

NAME TYPE REQUIRED DESCRIPTION
eventName string Yes Event name
eventDateTime string Yes Date / time for the event
eventLocationName string No Event location
eventSponsorName string No Event sponsor
durationInMinutes string No Event duration
comments string No Freeform comment
eventsType string No Type of event
invitationListID int Yes Associated Invitation List

Permitted field values

FIELD RESTRICTIONS
eventDateTime Correct format is "2018-01-01 23:12:59"
eventsType Valid values: "Meeting", "Training", "Enrollment", "Sales event", "Assembly"
durationInMinutes Valid values: "30", "60", "90", "120", "180", "240", "1440"
invitationListID Valid values: 0 or a valid Invitation List ID

JSON response

The following is an example Event ID returned from a successful call (Status Code = 200).

106

Errors

    "message": "Event Name is blank",
    "status_code": "400"

    "status_code": "400"
}
{

    "message": "Invitation List ID must be 0, if no list is to be associated",
    "status_code": "400"
}
{
    "error": "Bad Request",
    "message": "Invitation List ID must be numeric",
